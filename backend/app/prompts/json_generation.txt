RULE 2 — AGGREGATION IN COLUMNS:
When the Execution Plan specifies metrics, render them as aggregation expressions in "columns":
  COUNT metric:  "COUNT(table.primary_key) AS count"
  SUM metric:    "SUM(table.column) AS alias"
  AVG metric:    "AVG(table.column) AS alias"
  MAX metric:    "MAX(table.column) AS alias"
  MIN metric:    "MIN(table.column) AS alias"
Use the exact alias from the Execution Plan (e.g., "avg_salary", "total_payroll", "count").
NEVER use bare column names when the plan specifies aggregation.
IMPORTANT: For ranking queries where metric operation is empty or none (e.g., Top 10 salaries, Highest salary employee), do NOT aggregate! Use unaggregated columns directly in columns without MAX() or MIN() (e.g. "payroll.base_salary AS base_salary", "employees.first_name", "employees.last_name"). RULE 3 — TIME GRANULARITY IN GROUP BY:
When the Execution Plan specifies time_granularity OR group_by contains "month", "week", "quarter", "year", "day":
  - Use DATE_TRUNC('granularity', table.date_col) in both "columns" AND "group_by"
  - Example for "month" grouping on employees:
      columns: ["DATE_TRUNC('month', employees.hire_date) AS month", "COUNT(employees.id) AS count"]
      group_by: ["DATE_TRUNC('month', employees.hire_date)"]
  - Example for "quarter" grouping on payroll:
      columns: ["DATE_TRUNC('quarter', payroll.period_start) AS quarter", "SUM(payroll.base_salary) AS total_salary"]
      group_by: ["DATE_TRUNC('quarter', payroll.period_start)"]
  - Example for "week" grouping on attendance:
      columns: ["DATE_TRUNC('week', attendance.date) AS week", "SUM(attendance.hours_worked) AS total_hours"]
      group_by: ["DATE_TRUNC('week', attendance.date)"]
  - Never sort by the raw unaggregated date column when grouping by time! Set sort field to the time granularity alias (e.g. sort field: "month" or "quarter"), NOT the raw column.
  - Set time_granularity field to the granularity string: "month", "week", "quarter", "year", "day"
